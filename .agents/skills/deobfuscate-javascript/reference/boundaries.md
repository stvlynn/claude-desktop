# Boundaries & terminal nodes — what full-restoration restores, and what it deliberately doesn't

← Back to [SKILL.md](../SKILL.md). Read this when deciding whether a chunk is a
terminal node (npm leaf, oversized-local, external, faced boundary), when
creating or resolving anything under `restored/boundaries/`, or when choosing
`build-import-graph.ts` depth flags. Related: the npm-first rules for
`restored/vendor/*` in [vendor-npm.md](vendor-npm.md).

## Terminal-node kinds

Full-restoration mode makes three categories of dependencies _terminal nodes_ in the manifest. They are recorded so consumers can reference them, but never have a ledger entry, never get renamed, and never get a `<basename>/` workspace dir:

| Kind              | Trigger                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 | What we do                                                                                                                                                                                                                                                                                                          |
| ----------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `npm-leaf`        | Basename in `CHUNK_NAME_REGISTRY` (`clsx-XXXX.js`, `react-XXXX.js`, `react-is-XXXX.js`, `react-style-singleton-XXXX.js`, `react-colorful-XXXX.js`, `dotlottie-react-XXXX.js`, `dayjs-XXXX.js`, `katex-XXXX.js`, `pdf-XXXX.js`, `docx-preview-XXXX.js`, `stylis-XXXX.js`, `rough.esm-XXXX.js`, `cytoscape.esm-XXXX.js`, Graphlib/Dagre `graphlib-*` / package-body `dagre-*`, D3/Mermaid `src-BhkLFyc4` / `src-6yFswxVy` / `treemap-*` / `sankeyLinkHorizontal-*` / `arc-*` / `line-*` / `pie-*` / `step-*` / `monotone-*` / `min-*` / `defaultLocale-*` / `path-*` / `Tableau10-*`, `tslib.es6-XXXX.js`, `jotai-react-XXXX.js`, `marked.esm-XXXX.js`, `floating-ui.react-dom-XXXX.js`). | `polish.ts → resolve-npm-imports.ts` rewrites consumer imports to the bare specifier (`import clsx from "clsx"`); the chunk file is left untouched.                                                                                                                                                                 |
| `oversized-local` | Only with explicit `--max-lines N` when a local sibling exceeds the cap. **Quick/targeted mode**, not deep restoration.                                                                                                                                                                                                                                                                                                                                                                                                                                                                 | Parse imports/exports (bindings link up) but stop BFS at the file. Consumer import stays `import { xt as someAlias } from "./setting-storage-XXXX.js"` — the agent renames `someAlias` from consumer usage, not the producer.                                                                                       |
| `external`        | Bare specifier (`react`, `vite`) not seen as a chunk in the manifest.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   | Same as `npm-leaf`.                                                                                                                                                                                                                                                                                                 |
| `faced-boundary`  | A genuinely **third-party** vendor/runtime-dominated chunk (Zod's 308-export `src`, a host/`vscode-api` bridge, the Statsig SDK) imported by many features via cryptic aliases. **Project/feature chunks are never `faced`** — "big" or "imported by many" is not a facing reason; only vendor/runtime _content_ is.                                                                                                                                                                    | `make-facade.ts <chunk>` → typed facade; record under `dependencyBoundaryFacades`; `ledger.ts mark-faced <basename>` (refuses app-entry basenames without `--force`). A facade is an _open boundary_, not done — see [Boundary lifecycle](#boundary-lifecycle--three-terminal-states-never-a-permanent-any-facade). |

**Depth rule (recursion):** deep/full restoration recurses the **entire reachable project-local import graph**, not just the entry's direct imports — deep-restoring `app-main-*` pulls in `app-shell-*` and everything it imports, transitively to the leaves. An entry is **not `done` while any project-local chunk it transitively imports is still a facade or placeholder.** `build-import-graph.ts` enqueues every local sibling (default `--max-lines 0`); the loop must drain `ledger.ts frontier` to empty. Facing is reserved for genuine third-party vendor/runtime leaves (an _open boundary_ to report, never a way to call an app chunk "done"); a positive `--max-lines` is only for an explicit quick/targeted restore. Final proof is `quality-gate.ts <target-dir>` over the whole public target — not a boundary-only audit or import-map statuses.

## Boundary lifecycle — three terminal states, never a permanent `any`-facade

`restored/boundaries/*.ts` is **scaffolding in transit**, not a resting place. A generated `export declare const X: any` facade type-checks but is dead weight and hides whether a dep is third-party or unfinished app code. `scripts/chunk-classification.ts → classifyBoundary()` splits every boundary into two kinds (driven by the IMPORT_MAP `vendor` field — anything other than the literal `"runtime"` marker names a real npm family), each with a defined exit:

1. **Third-party npm package → bare re-export shim (DONE).** When the chunk _is_ a recognised package (`lodash`, `react-router`, `cmdk`, `@tanstack/react-form`, D3 diagram helpers such as `d3-axis`/`d3-selection`, `d3-array`, `d3-format`, `d3-path`, `d3-scale`, `d3-color`, `d3-interpolate`, `d3-hierarchy`/`d3-scale-chromatic`, `d3-sankey`, and `d3-shape`, `graphlib`, `dagre`, `pdfjs-dist`/PDF.js, `react-dom/client`, `react-intl`/FormatJS, `react-is`, `react-redux`, `react-style-singleton`, `use-sync-external-store`, `react-colorful`, `@lottiefiles/dotlottie-react`, `dayjs`, `katex`, `roughjs`, `cytoscape`, `cytoscape-fcose`, `cytoscape-cose-bilkent`, `jotai`, `@dnd-kit/*`, `framer-motion`, `mdast-util-*`, `@pierre/diffs`, `zod`, `@radix-ui/*`, Segment…), the deliverable is a thin re-export: `make-facade.ts <chunk> --reexport <specifier> [--export-star | --reexport-named a,b,c] [--name-map alias→real.json]`. Resolves to real `@types`, finished — `boundaries/highlight-js-core.ts` is the model. **Verify first:** the chunk must be _stock_, not a Codex fork (`@pierre/*` and other vendors are forked — see [codex-ref.md](codex-ref.md)). Use `ref/node_modules` / `ref/package.json` as supporting evidence when present, but **do not treat a missing extracted dependency as a veto**: extracted app bundles can be pruned/incomplete, and high-confidence packages already in `CHUNK_NAME_REGISTRY` or the Codex package table (for example Lodash current aggregate chunks, React Router, cmdk, TanStack React Form, Graphlib/Dagre, PDF.js, D3 diagram helpers, `react-dom/client`, `react-intl`/FormatJS, React companion packages including `react-redux`, `react-style-singleton`, react-colorful, dotLottie React, Day.js, KaTeX, RoughJS, Cytoscape, and Jotai) still become npm-backed re-export shims. If the restored tree lacks the dependency, add/keep an ambient declaration or record the package install requirement; do not hand-write a compatibility implementation. Consumers keep importing from `boundaries/`; for cryptic aliases (`appScopeC`), `--name-map` bridges `{ "<publicAlias>": "<realExport>" }`. `quality-gate.ts <target>` flags any vendor-npm boundary still an `any`-facade (`full-restoration-npm-boundary-not-resolved`) and public high-confidence npm shims that are not bare package re-exports (`third-party-npm-shim-not-reexport`).
   When a confirmed package-backed chunk has one or two legacy aliases that the
   current npm package does not expose (for example Mermaid's `curveBumpX` /
   `curveBumpY` next to `d3-shape@3.2.0`), keep those as tiny typed wrappers in a
   helper module under the npm-backed shim and re-export every real package API
   from npm. Do not fabricate imports from nonexistent package exports, and do
   not let the wrapper grow into a hand-written package body.

2. **Genuine app/host runtime → typed facade, or an opt-in runnable interim.** For the app's own runtime (`app-scope`, the `vscode`/host bridge, `rpc`, `host-config`, `product-logger`, persisted-signal state…), the typed `make-facade.ts` `any`-facade is the **type-check-only** interim (recorded under `dependencyBoundaryFacades` + `ledger.ts mark-faced`). To make the app **runnable before** the runtime is restored, `make-facade.ts <chunk> --passthrough <ref-relpath> [--name-map]` emits a `// @ts-nocheck` + `// TODO: deep-restore` module re-exporting the **original** ref chunk (makes `restored/` depend on the unrestored `ref/` tree — a marked stopgap, not a deliverable).

3. **Restore and move OUT of `boundaries/` (the only completion for kind 2).** A runtime facade/passthrough is an _open boundary_, never "done". Deep-restore, `ledger.ts set-organization` into its real domain (`host/`, `contexts/`, `utils/`, …), `promote-organized.ts` out of `boundaries/`. `quality-gate.ts <target-dir>` over the whole target — not a `boundaries/` grep — is the proof.

## `build-import-graph.ts` depth override flags

- `--max-lines 0` — default; disable the cap entirely (restore _everything_ project-local).
- `--max-lines 5000` — quick/targeted mode; skip local siblings above the cap.
- `--include foo-HASH,bar-HASH` — force-restore specific basenames even if they exceed the cap. Repeat the flag is fine; comma-separated also works.

The entry is _always_ exempt from the cap.
