# Quality bar — anti-patterns to refuse before declaring done

← Back to [SKILL.md](../SKILL.md). **Read this checklist before declaring any
restore done** — every tier for the naming/readability section, plus the deep
section when the deliverable is deep/typed/production.

## Both tiers — naming/readability (the default tier's only hard bar)

These produce "looks OK on a skim, falls apart on a real read" output — the hard bar in **every** tier, because they are about readability, not compilation:

- **Program-scope-only rename.** Top-level exports named (`AppShellTabContent`, `RightPanelTabs`) but bodies still `let k = useIntl(), [A, M] = useState(false), N = !h, …`. Cure: rename into function bodies ([stage-2-restore.md → Step 2.5](../stages/stage-2-restore.md#step-25--dont-stop-at-program-scope)) until single-letter density is low.
- **Generated fallback names shipped as "readable".** Code contains `ImportedBinding1`, `callbackValue1`, `localValue1`, `local274`, `argument8`, `elementNode1`, `hookValue1`, `restoredHelper1`, `param1`, `buttonValue3`, `buttonParam1`, `contextParam14`, or `DistO`. Cure: assign semantic names from producer/consumer usage, JSX shape, call graph, domain context; `quality-gate.ts --allow-flat` flags the common patterns.
- **React Compiler scaffolding left in load-bearing functions.** `let t = compilerCache.c(N), …; if (t[5] !== n …) { …compute…; t[5] = n; } else { … = t[5]; }` still present — hurts _reading_, fix in either tier. Cure: hand-strip the worst offenders (keep the compute branch, drop the cache slots).
- **Lowercase JSX component aliases left as HTML intrinsics.** JSX conversion produces `<b id="…" defaultMessage="…" />` where the lowercase tag is actually an imported component alias (`FormattedMessage`) — `jsx-runtime.ts` kept the minified binding, so React treats it as intrinsic. Cure: recover the component name from the producer/import map, rewrite tags to PascalCase.
- **Known npm vendor rewritten as a local compatibility implementation.** A public file such as `vendor/react-intl.tsx`, `vendor/react-style-singleton.ts`, `vendor/jotai-runtime.tsx`, `vendor/segment-analytics.ts`, `vendor/segment-analytics-integration.ts`, `vendor/roughjs.ts`, `vendor/cytoscape-runtime.ts`, `vendor/dayjs.ts`, `vendor/graphlib.ts`, `vendor/dagre.ts`, `vendor/pdfjs.ts`, `vendor/pdfjs-entry.ts`, `vendor/docx-preview.ts`, `vendor/stylis.ts`, `vendor/highlight-js-core.ts`, `vendor/entities-escape.ts`, or D3 shims like `vendor/d3-array-min.ts`, `vendor/d3-format-default-locale.ts`, `vendor/d3-path.ts`, `vendor/d3-tableau10.ts`, `vendor/d3-scale-band.ts`, `vendor/d3-scale-linear.ts`, `vendor/d3-interpolate-string.ts`, and `vendor/d3-interpolate-transform.ts` must not hand-implement a subset of the package API. Cure: add the npm dependency when needed and ship a thin bare re-export / alias shim with real package types. `quality-gate.ts` reports `third-party-npm-shim-not-reexport` for high-confidence package shims that do not re-export their npm package, including shims identified by provenance source chunk (for example FormatJS `lib-BWT6A3Q0` → `react-intl`; Highlight.js `core-DMiaGTKr` → `highlight.js/lib/core`; RoughJS `rough.esm-BmcJJgrn` → `roughjs`; Cytoscape `cytoscape.esm-gCnb3XbU` → `cytoscape`; Graphlib `graphlib-*` → `graphlib`; package-body Dagre `dagre-BqhzN4_p` / `dagre-Ba2O9HBx` / `dagre-5oTtyBe6` → `dagre`; PDF.js `pdf-CaT3Fa-k` / `pdf-CEc0zmuq` → `pdfjs-dist`; Docx Preview `docx-preview-G1XqxLZP` → `docx-preview`; Stylis `stylis-CNWqMcUo` → `stylis`; sanitize-url `dist-CD74BDfk` / `dist-BO3qvbGN` / `dist-Kh8ku4yn` / `dist-WEF1yh7f` → `@braintree/sanitize-url`; Segment AJS destination `ajs-destination-*` → `@segment/analytics-next/dist/cjs/plugins/ajs-destination`; D3 chunks `min-*` → `d3-array`, `defaultLocale-*` → `d3-format`, `path-*` → `d3-path`, `Tableau10-*` → `d3-scale-chromatic`, `band-*` / `linear-*` → `d3-scale` plus `d3-array`, and `string-*` → `d3-color` plus `d3-interpolate`) even if the public filename changes, by vendor-file API fingerprints (for example exported `useIntl`/`FormattedMessage`/`IntlProvider` implies `react-intl`; exported `styleHookSingleton`/`styleSingleton`/`stylesheetSingleton` implies `react-style-singleton`; exported `getDocument`/`GlobalWorkerOptions`/`AnnotationLayer`/`TextLayer` implies `pdfjs-dist`; exported `sanitizeUrl` implies `@braintree/sanitize-url`; exported `AnalyticsBrowser`/`segmentio` implies `@segment/analytics-next`; exported `ajsDestinations` implies `@segment/analytics-next/dist/cjs/plugins/ajs-destination`; exported `roughjs` with `canvas`/`svg`/`generator` implies `roughjs`; exported `interpolateTransformCss`/`interpolateTransformSvg` implies `d3-interpolate`), and by a package-name fallback: if `vendor/<name>.ts[x]` matches a dependency declared in the nearest `package.json`, it must re-export from `<name>` instead of implementing a local subset. It reports `third-party-npm-shim-dependency-missing` when the shim exists but the nearest `package.json` does not declare the package root.
- **Mechanical/batch checkpoint promoted into `restored/`.** `auto-restore-full.ts`/batch-`polish.ts` output, hash-basename files (`button-bq66r8jD.tsx`), or `--write-target-checkpoints` files directly in `restored/` — skipping staging → organize → promote ([Output conventions](../SKILL.md#output-conventions)). Cure: keep batch/script output in the `.deobfuscate-javascript/` staging tree; promote only after a file meets the promotion bar.
- **Bundle body merely moved/renamed into a semantic folder.** A `vendor/*-bundle.ts` body copied to `settings/settings-page-current.tsx`, `app-main/current-*-backing.ts`, `worktree/*-backing.ts`, etc. with a header like `implementation/backing with restored dependency imports`, while still thousands of lines of minified exports, compat-slot aliases, `__vite__mapDeps`, or React Compiler residue. That is a WIP relocation, not a restore. Cure: split the body into maintainable semantic modules, replace npm/vendor parts with bare package imports or real boundary re-exports, keep only a small barrel/shim at the old path, and run `quality-gate.ts` on the semantic modules. `quality-gate.ts` reports `relocated-bundle-body` for this anti-pattern; `--allow-flat` is not a completion waiver.
- **Mechanical checkpoints built, but `restored/` left empty / nothing organized.** `_full/checkpoints/` full but `restored/` has no promoted tree — declared "done" at the checkpoint. Hard failure in **both** tiers. Cure: `plan-organize.ts` → review/`--apply` → `promote-organized.ts` until `ledger.ts frontier --stage promote` is empty and `quality-gate.ts <target-dir>` exits 0 (gate errors `full-restoration-checkpoints-not-drained` / `-organize-incomplete`). See [full-restoration.md → Step 4](../workflows/full-restoration.md).

## Deep / production mode only

Typing/structure/compilability requirements — **not** failures in the default readable tier (a well-named untyped file is a valid default deliverable). They apply only for deep/typed/production output, enforced by [Stage 3's D0 gate](../stages/stage-3-finalize.md#d0--did-the-rename-and-split-actually-finish) and [acceptance review](../stages/stage-3-finalize.md):

- **Multi-export bundle delivered as a single file.** Polished `.tsx` > 1 000 lines with a `export const Foo = { Name1, Name2, … }` registry at `<target-dir>/<bundle>.tsx` not `<bundle>/`. Cure: [multi-export-bundle.md](../workflows/multi-export-bundle.md). (Default tier may leave it flat.)
- **Compatibility shim confused with body restoration.** Replacing `vendor/foo-bundle.ts` with `export * from "../domain/foo-backing"` is only acceptable as an intermediate once the target body is actively being split; it does not satisfy deep restore unless `../domain/foo-*` is already semantic, typed, and split enough to pass acceptance. A shim-only commit is an alias-map change, not restored code.
- **Checkpoint shipped as deep completion.** Copying `$WS/polished.tsx`, `_full/checkpoints/<basename>.tsx`, or `auto-restore-full.ts` output to the target and calling the _deep_ restore done ships the mechanical checkpoint. Cure: semantically rewrite, semantic final filenames, script pre-filter, then Stage 3 acceptance review.
- **Polished `.js` instead of typed `.tsx` (deep mode).** Stage 3's D5 requires a `.tsx` with a real `Props` interface (or type alias) on every exported component, param/return types where helpful, and `ReactNode` / `ReactElement` / event-handler types for JSX slots. `function MyRow(props)` untyped is fine in the **default** tier; deep-mode fail only. See [D5](../stages/stage-3-finalize.md#d5--typescript-types-and-the-tsx-recipe).
- **Pure icon module shipped with bundle residue.** One-SVG file still has `jsx-runtime`, `__toESM(loadJsxRuntimeRaw())`, `var jsxRuntime = …`, or untyped `export const i = props => <svg …>`. Deep cure: `semantic-finalize.ts --recipe icon` → `SVGProps<SVGSVGElement>`, `IconProps`, semantic `*Icon` name, default export.
- **Small multi-icon chunk left as one file.** `expand-XXXX.tsx` exports ≥2 independent SVG icons flat. Deep cure: split into `<basename>/types.ts`, one kebab `*-icon.tsx` per icon (`expand-icon.tsx` exporting `ExpandIcon`), and `<basename>/index.ts` barrel.
- **Reusable UI component left mechanically renamed (deep mode).** Button/control still has `s`, `c`, `D`, `O`, `<spinnerT>`, untyped `props`, or lookup tables without `as const`. Deep cure: `semantic-finalize.ts --recipe button` — semantic lookup-table names, `keyof typeof` unions, DOM props interface, `forwardRef`, `displayName`, default export. (Default tier: meaningful local names + clear JSX suffice.)
- **Deep request downgraded to quick mode.** On "deep"/"full"/"完整"/"深度"/include-referenced-chunks, running `build-import-graph.ts` with a positive `--max-lines` then skipping `oversized-local` chunks is a fail. Use the default no-cap graph, or explicitly say you're producing a partial restore.
- **App/feature chunk faced instead of recursively restored (deep/full mode).** A referenced project-local chunk (`app-shell-*`, a feature page/panel/view, a context or hook) left as a `boundaries/*.ts` facade/placeholder and marked `boundary`/`faced` while the restore is called complete — treating "large"/"imported by many" as a facing reason, or stopping recursion at direct imports. Cure: follow the graph to the leaves; every reachable project-local feature chunk must reach `done`, not `faced`; drain `ledger.ts frontier` to empty first. Facing is valid **only** for genuine third-party vendor/runtime leaves (Zod, Statsig, host/runtime bridge), and even then an _open boundary_ to report (`ledger.ts mark-faced` refuses app-entry basenames without `--force`). A boundary that's a **known npm package** is resolved to a bare re-export shim (`make-facade.ts --reexport`), not faced — see [Boundary lifecycle](boundaries.md#boundary-lifecycle--three-terminal-states-never-a-permanent-any-facade).
- **Full/deep completion inferred from a narrow audit.** `restored/boundaries` has few placeholders, or `IMPORT_MAP.json` says many chunks are `done`, so the restore is declared complete while `composer-*`, `local-conversation-*`, `thread-*`, `review-*`, `*-page`/`*-panel` are still mechanical. Cure: `quality-gate.ts <target-dir>` over the whole target — the coverage gate reads the manifest + legacy/current import maps and refuses app-feature `@ts-nocheck`, `export declare const`, empty `export {}`, `mechanical-readable-restored`, unresolved `oversized-local`, and missing Stage 3 acceptance evidence.
- **Deep restore shipped without the Stage 3 acceptance review.** `promote-final.ts` exited 0, files copied, deep task declared done — but no one read them end-to-end. Script gates miss grammatical-but-meaningless names, missing `Props`/`forwardRef`/`displayName`/`as const`, untyped `props`. Only an end-to-end read against the quality bar (host self-review, or optional sub-agent) accepts a deep deliverable; "no sub-agent" is never a reason to skip — the host reads. (Default tier needs no LOOP; optional naming-only self-review is enough.)
